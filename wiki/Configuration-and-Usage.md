# Настройка и использование

После запуска контейнера (см. **[Быстрый старт](Quick-Start)**) основная работа ведётся через
веб-интерфейс TorrentMonitor.

> 📖 Эта вики описывает **Docker-контейнер**. Работа с самим приложением — веб-интерфейс,
> настройки, поддерживаемые трекеры, подключение торрент-клиентов, уведомления — описана в
> **[Вики TorrentMonitor](https://github.com/ElizarovEugene/TorrentMonitor/wiki)**.

---

## Первый вход

1. Откройте в браузере адрес контейнера, например <http://localhost:8080> (или
   `http://IP-сервера:8080`).
2. При первом запуске приложение создаёт базу данных и файл настроек в томе `/data/htdocs/db`.
3. Дальнейшая настройка приложения (торрент-клиент, трекеры, раздачи для мониторинга) — по
   инструкциям из **[Вики TorrentMonitor](https://github.com/ElizarovEugene/TorrentMonitor/wiki)**.

> 🔐 TorrentMonitor не предназначен для прямого доступа из интернета без защиты. Если нужен внешний
> доступ, ставьте контейнер за обратный прокси (Nginx Proxy Manager, Traefik, Caddy) с HTTPS и
> аутентификацией.

---

## Подключение торрент-клиента из контейнера

Сама настройка клиента выполняется в веб-интерфейсе приложения (см.
**[Вики TorrentMonitor](https://github.com/ElizarovEugene/TorrentMonitor/wiki)**). При работе в
Docker учитывайте особенности адресации:

- **Клиент на хосте** — в обычном (bridge) режиме сети указывайте IP-адрес хоста, а не
  `localhost`/`127.0.0.1`: изнутри контейнера `localhost` — это сам контейнер. В Docker Desktop
  можно использовать `host.docker.internal`.
- **Клиент в соседнем контейнере** — поместите оба контейнера в общую сеть Docker и обращайтесь к
  клиенту по имени контейнера (например, `transmission`).
- **Общее сетевое пространство** — если контейнер запущен с `network_mode: host` либо делит сетевой
  стек с клиентом (`network_mode: service:vpn` / `container:...` — типично для связок через VPN),
  клиент доступен по `localhost`.
- Пример готовой связки нескольких контейнеров — на странице
  **[TorrentMonitor + TOR + Transmission](TOR-and-Transmission)**.

## Расписание проверок

Частота проверки трекеров задаётся переменной окружения
**[CRON_TIMEOUT](Environment-Variables#cron_timeout--расписание-проверок)** при запуске контейнера,
а не в веб-интерфейсе. По умолчанию — раз в час.

Чтобы запустить проверку вручную, не дожидаясь расписания, можно выполнить команду движка внутри
контейнера:

```bash
docker exec torrentmonitor php -q /data/htdocs/engine.php
```

## Автообновление приложения

В веб-интерфейсе TorrentMonitor есть функция самообновления (обновление кода самого приложения).

Обновлять приложение лучше через обновление Docker-образа — см.
**[Обновление и обслуживание](Updating-and-Maintenance)**.

## Где хранятся данные

| Что | Где внутри контейнера |
| --- | --- |
| База данных и настройки | `/data/htdocs/db` |
| Скачанные `.torrent`-файлы | `/data/htdocs/torrents` |
| Логи (опционально) | `/data/htdocs/logs` |

Эти пути выносятся в тома — подробнее на странице **[Тома и постоянные данные](Volumes-and-Data)**.
